Cursor 配置指南

本指南将帮助您在 Cursor 编辑器中配置和使用"天机"思维模型 MCP 服务器，让您在编码过程中轻松访问强大的思维工具。Cursor 原生支持 MCP 协议，配置简单直观。

Cursor MCP 思维模型 AI 助手

配置步骤概览

1

安装 Cursor 编辑器

如果您尚未安装 Cursor，请先下载并安装：

下载 Cursor

Cursor 是一个基于 VS Code 的编辑器，具有强大的 AI 功能，同时原生支持 MCP 协议，适合与"天机"思维模型服务器配合使用。

2

启动"天机"MCP 服务器

确保"天机"思维模型 MCP 服务器已正确安装并且正在运行：

# 如果使用全局安装
thinking-models-server

# 如果使用本地安装
npm run start-thinking-models

服务器启动后，应该会显示一条消息，表明服务器已经在监听连接。

3

配置 Cursor MCP 设置

在 Cursor 中配置 MCP 服务器：

  1. 点击左下角的设置图标（⚙️）或按下 Ctrl+, 打开设置
  2. 搜索"MCP"或导航至"AI"设置部分
  3. 找到"MCP: Servers"设置选项
  4. 点击"Edit in settings.json"编辑配置文件

在 settings.json 中添加以下配置：

{
  // ... 其他设置 ...
  
  "mcp.servers": [
    {
      "name": "天机思维模型服务器",
      "command": "thinking-models-server",
      "address": null,
      "description": "思维模型MCP服务器",
      "status": "stopped"
    }
  ],
  
  // 如果希望默认使用天机服务器，可以添加：
  "mcp.defaultServer": "天机思维模型服务器"
}
如果您使用本地项目安装或自定义路径，请将"command"设置为正确的启动命令，例如：npm run start-thinking-models 或完整路径。

4

连接到"天机"服务器

配置完成后，需要连接到服务器：

方法一：通过命令面板

  1. 按下 Ctrl+Shift+P 打开命令面板
  2. 输入"MCP: Connect to Server"并选择
  3. 从列表中选择"天机思维模型服务器"

方法二：通过状态栏

  1. 在 Cursor 底部状态栏找到 MCP 相关图标
  2. 点击图标，从弹出的菜单中选择"天机思维模型服务器"
  3. 选择"Connect"进行连接

连接成功指示

连接成功后，Cursor 将显示通知，状态栏 MCP 图标会变成绿色，并显示"已连接到天机思维模型服务器"。

5

使用"天机"思维模型

成功连接后，可以开始在 Cursor 中使用"天机"思维模型工具：

使用 Cursor 聊天面板

  1. 点击左侧活动栏中的聊天图标，或使用快捷键 Ctrl+I 打开聊天面板
  2. 在聊天面板中输入与思维模型相关的查询，例如：
    • "天机"中有哪些决策类思维模型？
    • 请使用SWOT分析模型评估一个新产品上市
    • 我需要一个框架来分析复杂系统，有什么推荐？
提示： Cursor 的智能度高于普通编辑器，可以直接在聊天面板中使用自然语言与"天机"服务器交互，获取思维模型推荐或分析问题。请参考工具文档了解更多使用方法。

在文档内使用内联提示

Cursor 允许在代码或文档中直接使用内联提示：

  1. 在代码或文档中输入注释形式的提示，例如：
// 使用天机的系统思维分析以下问题：
// 我们的应用需要优化用户留存率，当前数据表明...
  1. 选中提示文本，按 Ctrl+Enter 或 Alt+\\
  2. Cursor 将通过"天机"服务器处理提示并在下方生成回应

Cursor 的高级配置

切换多个 MCP 服务器

Cursor 支持配置多个 MCP 服务器，方便在不同场景切换使用：

{
  "mcp.servers": [
    {
      "name": "天机思维模型服务器",
      "command": "thinking-models-server",
      "address": null,
      "description": "思维模型MCP服务器"
    },
    {
      "name": "其他MCP服务器",
      "command": "其他服务器命令",
      "address": null,
      "description": "其他MCP服务器"
    }
  ]
}

使用命令面板中的"MCP: Connect to Server"命令可以快速在多个服务器间切换。

使用网络模式连接

如果您更喜欢使用网络模式而非命令行模式，可以这样配置：

  1. 使用网络模式启动"天机"服务器：
# 设置环境变量并启动
THINKING_MODELS_PORT=3456 thinking-models-server
  1. 在 Cursor settings.json 中配置：
{
  "mcp.servers": [
    {
      "name": "天机思维模型服务器",
      "command": null,
      "address": "localhost:3456",
      "description": "思维模型MCP服务器(网络模式)"
    }
  ]
}

自定义快捷键

为快速访问"天机"功能，配置专用快捷键：

  1. 打开键盘快捷键设置（Ctrl+K Ctrl+S）
  2. 搜索"MCP"相关命令
  3. 为"MCP: Connect to Server"等命令设置自定义快捷键

常见问题与排查

可能的原因和解决方案：

  • 服务器未启动：确保在单独的终端窗口中启动了"天机"服务器
  • 命令路径错误：确保服务器命令在系统路径中，或提供完整路径
  • 配置错误：检查 settings.json 中的服务器配置是否正确

尝试使用"MCP: Show Server Status"命令查看服务器状态和错误信息。

可能的原因和解决方案：

  • 服务器内部错误：检查服务器终端输出是否有错误信息
  • 查询格式：尝试使用更简单明确的查询格式
  • 冲突设置：检查 Cursor 的 AI 设置是否有其他冲突配置

尝试重新连接服务器或重启 Cursor 解决临时问题。

可能的原因和解决方案：

  • Cursor 版本：确保使用最新版本的 Cursor，旧版可能不支持 MCP
  • 设置搜索：尝试在设置中搜索"Model Context Protocol"或直接编辑 settings.json
  • 重启问题：某些设置更改需要重启 Cursor 才能生效

访问 Cursor 官方文档 了解更多关于 MCP 支持的详细信息。

Cursor 与思维模型的优势

  • 可以在编码过程中直接应用系统思维、解决方案架构等高层次思考框架
