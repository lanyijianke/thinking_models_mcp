Visual Studio Code 配置指南

本指南将帮助您在 VS Code 中配置和使用"天机"思维模型 MCP 服务器，通过 MCP (Model Context Protocol) 协议连接，让您在编码过程中轻松访问思维模型工具。

VS Code MCP 思维模型 AI 助手

配置步骤概览

1

安装必要的 VS Code 扩展

首先需要安装 MCP 集成扩展，提供 VS Code 与"天机"服务器之间的通信桥梁。

Copilot MCP

管理和连接 MCP 服务器的 VS Code 扩展

在 Marketplace 查看

Cline 或其他 AI 助手

支持 MCP 的 AI 助手（可选）

在 Marketplace 查看

在 VS Code 中安装扩展的方法：

  1. 点击活动栏中的扩展图标（或按下 Ctrl+Shift+X）
  2. 在搜索框中输入"Copilot MCP"
  3. 找到上述扩展并点击"安装"按钮

2

启动"天机"MCP 服务器

确保"天机"思维模型 MCP 服务器已正确安装并且正在运行：

# 如果使用全局安装
thinking-models-server

# 如果使用本地安装
npm run start-thinking-models

服务器启动后，应该会显示一条消息，表明服务器已经在监听连接。

3

配置 Copilot MCP 扩展

安装完成后，需要配置 Copilot MCP 扩展连接到"天机"服务器：

  1. 点击 VS Code 状态栏中的 Copilot MCP 图标（或在命令面板中搜索"Copilot MCP"）
  2. 选择"Add Server"添加新服务器
  3. 对于本地运行的"天机"服务器，配置如下：
配置项 值 说明
名称 Tianji MCP 服务器的显示名称
命令行 thinking-models-server 启动服务器的命令（全局安装）
服务器类型 stdio 通过标准输入/输出与客户端通信
如果您使用本地项目安装或自定义路径，请将命令行设置为正确的启动命令，例如：npm run start-thinking-models 或完整路径。

4

启动并连接服务器

配置完成后，需要启动并连接到服务器：

  1. 在 VS Code 状态栏中点击 Copilot MCP 图标
  2. 从下拉列表中选择刚刚添加的"Tianji MCP"服务器
  3. 点击"Connect"进行连接

如果连接成功，状态栏图标会变为绿色，同时会显示"Tianji MCP"的名称。

连接成功指示

当连接成功时，VS Code 将显示通知"连接到 Tianji MCP 成功"，并且状态栏会显示连接状态指示器。

5

使用"天机"思维模型

成功连接后，可以开始使用"天机"思维模型工具了：

方法一：通过 VS Code 聊天界面使用

  1. 点击 VS Code 左侧活动栏中的聊天图标（如果已安装 GitHub Copilot、Cline 或其他支持聊天的 AI 插件）
  2. 在聊天框中输入与思维模型相关的查询，如：
    • "天机"当前版本号是多少？
    • 请列出所有与决策相关的思维模型
    • 我正在分析一个复杂问题，需要一个系统思维的框架，有什么推荐？

方法二：通过 Copilot MCP 命令面板使用

  1. 按下 Ctrl+Shift+P 打开命令面板
  2. 输入"Copilot MCP"查看可用命令
  3. 选择"Send Command to Server"向服务器发送命令
  4. 在输入框中输入您的查询
提示： 要充分利用"天机"的功能，请查阅工具文档，了解各类思维工具的使用方法和示例。

高级配置选项

使用网络模式连接

如果您更喜欢使用网络连接模式而非标准输入/输出模式，可以按以下步骤配置：

  1. 使用网络模式启动"天机"服务器：
# 设置环境变量并启动
THINKING_MODELS_PORT=3456 thinking-models-server
  1. 在 Copilot MCP 中添加服务器时配置：
配置项 值
服务器类型 network
地址 localhost:3456

自动启动服务器

您可以配置 VS Code 在启动时自动连接到"天机"服务器：

  1. 打开 VS Code 设置（Ctrl+,）
  2. 搜索"Copilot MCP"
  3. 找到"Auto Connect Server"选项并启用
  4. 设置"Default Server"为您的"Tianji MCP"配置

快捷键配置

为快速访问"天机"功能，可以设置自定义快捷键：

  1. 打开 VS Code 键盘快捷键（Ctrl+K Ctrl+S）
  2. 搜索"Copilot MCP"
  3. 为常用命令（如"Send Command to Server"）设置快捷键

常见问题与排查

可能的原因和解决方案：

  • 服务器未启动：确保您已在单独的终端窗口中启动了"天机"服务器
  • 命令路径错误：确认服务器命令在您的系统环境变量中，或提供完整路径
  • 端口冲突：如使用网络模式，确保指定的端口未被占用

尝试在终端中手动运行服务器命令，查看是否有错误提示。

可能的原因和解决方案：

  • 服务器异常：检查服务器终端输出是否有错误信息
  • 命令格式：确保您的请求格式符合服务器要求
  • 超时：大型请求可能需要更长处理时间，请耐心等待

尝试发送一个简单的请求，如"天机"当前版本号是多少？测试连接状态。

可能的原因和解决方案：

  • 配置未保存：确认您已正确添加并保存了服务器配置
  • 扩展版本：确保 Copilot MCP 扩展是最新版本
  • VS Code 重启：有时候需要重启 VS Code 使配置生效

尝试重新添加服务器配置，或卸载并重新安装 Copilot MCP 扩展。
