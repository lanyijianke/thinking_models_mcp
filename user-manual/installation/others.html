其他客户端配置指南

除了 VS Code、Cursor 和 Claude 桌面应用外，"天机"思维模型 MCP 服务器还可以与多种其他支持 Model Context Protocol (MCP) 的客户端配合使用。本指南将介绍这些客户端的配置方法。

MCP 客户端 思维模型 集成指南

通用 MCP 客户端配置原则

适用于所有支持 MCP 协议的客户端

无论您使用哪种 MCP 客户端，配置"天机"思维模型服务器的基本原则都是相似的。了解这些通用原则，可以帮助您在任何支持 MCP 的客户端中快速设置"天机"。

MCP 连接模式

"天机"服务器支持两种主要的连接模式：

连接模式 描述 配置参数 适用场景
标准输入/输出 (stdio) 客户端通过启动服务器命令，直接与其标准输入/输出通信 服务器命令：thinking-models-server 本地开发环境，无需网络通信的场景
网络模式 (network) 服务器在指定端口监听，客户端通过网络连接通信 地址：localhost:PORT 或 IP:PORT 远程连接，多客户端共享一个服务器实例

通用配置步骤

  1. 确保"天机"思维模型 MCP 服务器已安装并运行
  2. 在客户端中找到 MCP 服务器或扩展配置部分
  3. 添加新服务器，根据选择的连接模式提供命令或地址
  4. 连接服务器并验证连接是否成功
  5. 尝试发送基本查询测试功能

通用验证查询

可以在任何 MCP 客户端中使用以下查询验证"天机"服务器连接是否正常：

// 检查服务器状态
请使用get-server-version工具获取"天机"服务器的当前版本和状态。

// 验证模型数量
请使用count-models工具统计当前系统中的思维模型数量。

// 测试模型分类查询
请使用get-categories工具获取所有思维模型分类。
提示： 在任何新的客户端配置完成后，建议先使用上述验证查询测试连接和基本功能，确认无误后再进行更复杂的思维模型应用。

JetBrains IDE 配置

IntelliJ IDEA、PyCharm、WebStorm 等

JetBrains 系列 IDE 通过 AI Assistant 插件提供了 MCP 支持，可以配置连接到"天机"思维模型服务器。

前置条件

  1. 安装最新版本的 JetBrains IDE
  2. 安装 JetBrains AI Assistant 插件（确保版本支持 MCP）
  3. "天机"思维模型 MCP 服务器已安装并处于运行状态

配置步骤

  1. 打开 JetBrains IDE 的设置（File > Settings 或 Ctrl+Alt+S）
  2. 导航到 Tools > AI Assistant > Model Context Protocol
  3. 点击"Add Server"按钮
  4. 填写以下信息：
    • Name: 天机思维模型服务器
    • Command: thinking-models-server（如果使用命令行模式）
    • URL: localhost:3456（如果使用网络模式，端口根据实际配置填写）
  5. 确认并保存设置
  6. 在 AI Assistant 工具窗口中，从服务器下拉菜单选择"天机思维模型服务器"
  7. 连接到服务器并开始使用

使用方式

在 JetBrains IDE 中有多种方式使用"天机"思维模型：

  • AI Assistant 工具窗口：直接在对话框中输入查询
  • 内联提示：在代码中选中相关内容，右键菜单选择 AI Assistant 相关选项
  • 意图操作：使用 Alt+Enter 调出意图操作菜单，选择 AI 相关功能

Neovim 配置

通过 MCP 插件支持"天机"服务器

Neovim 用户可以通过安装支持 MCP 的插件来连接"天机"思维模型服务器。

前置条件

  1. Neovim 0.8.0 或更高版本
  2. 适当的包管理器（如 packer.nvim, vim-plug 等）
  3. Node.js 和 npm（部分插件依赖）

配置步骤

-- 使用 Packer 安装支持 MCP 的 AI 插件
-- 添加到您的插件配置文件 (通常是 init.lua 或类似文件)
use {
  'github-user/mcp-neovim-plugin',
  requires = {'nvim-lua/plenary.nvim'},
  config = function()
    require('mcp').setup({
      servers = {
        {
          name = "tianji",
          cmd = "thinking-models-server",
          -- 如果使用网络模式，则设置:
          -- host = "localhost",
          -- port = 3456
        }
      },
      default_server = "tianji",
      -- 其他可能的配置...
    })
  end
}

-- 配置快捷键
vim.keymap.set('n', 'ai', ':MCPPrompt', {noremap = true, silent = true})
vim.keymap.set('v', 'ai', ':MCPSelection', {noremap = true, silent = true})
注意： 由于 Neovim 生态系统中的 MCP 支持仍在发展中，上述代码是通用示例。请参考您选用的具体插件文档获取最新的配置指南。常见的支持 MCP 的 Neovim 插件包括 mcp.nvim、llm.nvim（带 MCP 扩展）等。

使用方式

  • 通过配置的快捷键调用 MCP 提示或选择命令
  • 在命令模式中使用 :MCPPrompt 打开提示输入
  • 在视觉模式中选择代码，然后使用 :MCPSelection 提交选中内容

Obsidian 配置

知识管理工具的思维模型集成

Obsidian 是一个强大的知识管理工具，通过支持 MCP 的社区插件，可以与"天机"思维模型服务器集成，实现笔记与思维工具的融合。

前置条件

  1. Obsidian 最新版本
  2. 已启用社区插件功能
  3. 安装支持 MCP 的插件（如 "MCP Assistant" 或类似插件）

配置步骤

  1. 打开 Obsidian 设置
  2. 找到已安装的 MCP 插件设置
  3. 添加新的 MCP 服务器：
    • 名称: 天机思维模型服务器
    • 类型: Local Command 或 Network
    • 命令: thinking-models-server（如果选择 Local Command）
    • URL: http://localhost:3456（如果选择 Network）
  4. 保存设置并启用服务器连接

使用场景

在 Obsidian 中使用"天机"思维模型的几种方式：

  • 思维框架生成：根据笔记内容自动推荐适合的思维模型
  • 笔记结构化：使用思维模型对笔记进行结构化组织
  • 问题解析：在研究复杂问题时应用系统思维框架
  • 知识图谱增强：使用思维模型对现有笔记进行关联分析
应用提示： Obsidian 与"天机"结合特别适合构建"第二大脑"系统。可以使用"天机"的思维框架对知识进行分类、关联和深化，形成更有结构的知识网络。尝试创建一个专门的"思维模型库"笔记，通过"天机"获取各类思维模型的详细描述，然后在日常笔记中引用这些模型。

命令行客户端

直接通过终端使用思维模型

对于命令行爱好者，可以通过简单的命令行工具直接与"天机"思维模型服务器进行交互，无需图形界面。

使用 curl 进行简单请求

如果"天机"服务器以网络模式运行，可以直接使用 curl 发送请求：

# 基本格式
curl -X POST http://localhost:3456/mcp/request \
  -H "Content-Type: application/json" \
  -d '{
    "id": "req1",
    "method": "invoke_tool",
    "params": {
      "name": "get-categories",
      "parameters": { "lang": "zh" }
    }
  }'

# 获取服务器版本
curl -X POST http://localhost:3456/mcp/request \
  -H "Content-Type: application/json" \
  -d '{
    "id": "req2",
    "method": "invoke_tool",
    "params": {
      "name": "get-server-version",
      "parameters": {}
    }
  }'

自定义集成

为现有应用添加"天机"支持

对于开发者，可以通过 MCP 协议为自己的应用程序添加"天机"思维模型支持。

Model Context Protocol 基础

MCP 协议是一个简单的 JSON-RPC 格式协议，可以通过多种方式实现：

  1. 标准输入/输出交互：启动服务器进程并通过标准 I/O 通信
  2. HTTP/WebSocket 网络通信：通过网络接口发送 HTTP 请求

实现 MCP 客户端

以下是使用不同编程语言实现简单 MCP 客户端的示例代码：

// JavaScript MCP 客户端示例
const fetch = require('node-fetch');

class MCPClient {
  constructor(serverUrl = 'http://localhost:3456') {
    this.serverUrl = serverUrl;
    this.reqId = 1;
  }
  
  async invokeTool(toolName, parameters) {
    const requestBody = {
      id: `req_${this.reqId++}`,
      method: 'invoke_tool',
      params: {
        name: toolName,
        parameters: parameters || {}
      }
    };
    
    const response = await fetch(`${this.serverUrl}/mcp/request`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(requestBody)
    });
    
    return await response.json();
  }
}

// 使用示例
async function example() {
  const client = new MCPClient();
  
  // 获取思维模型分类
  const categories = await client.invokeTool('get-categories', { lang: 'zh' });
  console.log('思维模型分类:', categories);
  
  // 推荐模型
  const models = await client.invokeTool('recommend-models-for-problem', {
    problem_keywords: ['决策', '不确定性'],
    limit: 3,
    lang: 'zh'
  });
  console.log('推荐模型:', models);
}

example().catch(console.error);
# Python MCP 客户端示例
import json
import requests

class MCPClient:
    def __init__(self, server_url='http://localhost:3456'):
        self.server_url = server_url
        self.req_id = 1
        
    def invoke_tool(self, tool_name, parameters=None):
        if parameters is None:
            parameters = {}
            
        request_body = {
            "id": f"req_{self.req_id}",
            "method": "invoke_tool",
            "params": {
                "name": tool_name,
                "parameters": parameters
            }
        }
        self.req_id += 1
        
        response = requests.post(
            f"{self.server_url}/mcp/request",
            headers={"Content-Type": "application/json"},
            json=request_body
        )
        
        return response.json()

# 使用示例
def main():
    client = MCPClient()
    
    # 获取服务器版本
    version_info = client.invoke_tool('get-server-version')
    print("服务器版本:", json.dumps(version_info, indent=2, ensure_ascii=False))
    
    # 统计模型数量
    count_info = client.invoke_tool('count-models', {'lang': 'zh'})
    print("模型统计:", json.dumps(count_info, indent=2, ensure_ascii=False))
    
if __name__ == "__main__":
    main()
// Java MCP 客户端示例
import com.fasterxml.jackson.databind.ObjectMapper;
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;
import java.util.HashMap;
import java.util.Map;
import java.util.concurrent.atomic.AtomicInteger;

public class MCPClient {
    private final String serverUrl;
    private final HttpClient client;
    private final ObjectMapper mapper;
    private final AtomicInteger reqId = new AtomicInteger(1);
    
    public MCPClient(String serverUrl) {
        this.serverUrl = serverUrl;
        this.client = HttpClient.newHttpClient();
        this.mapper = new ObjectMapper();
    }
    
    public Map invokeTool(String toolName, Map parameters) throws Exception {
        if (parameters == null) parameters = new HashMap<>();
        
        Map params = new HashMap<>();
        params.put("name", toolName);
        params.put("parameters", parameters);
        
        Map requestBody = new HashMap<>();
        requestBody.put("id", "req_" + reqId.getAndIncrement());
        requestBody.put("method", "invoke_tool");
        requestBody.put("params", params);
        
        String requestJson = mapper.writeValueAsString(requestBody);
        
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create(serverUrl + "/mcp/request"))
            .header("Content-Type", "application/json")
            .POST(HttpRequest.BodyPublishers.ofString(requestJson))
            .build();
            
        HttpResponse response = client.send(request, HttpResponse.BodyHandlers.ofString());
        return mapper.readValue(response.body(), Map.class);
    }
    
    public static void main(String[] args) {
        try {
            MCPClient client = new MCPClient("http://localhost:3456");
            
            // 获取模型分类
            Map params = new HashMap<>();
            params.put("lang", "zh");
            Map result = client.invokeTool("get-categories", params);
            
            System.out.println("思维模型分类: " + result);
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
