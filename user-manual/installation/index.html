本指南将帮助您快速设置并开始使用"天机"思维模型 MCP 服务器。"天机"采用模型上下文协议（Model Context Protocol, MCP）作为通信标准，可与多种支持 MCP 的客户端配合使用。

安装步骤概览

1

系统要求

确认您的系统满足以下最低要求：

  • Node.js 16.x 或更高版本
  • NPM 7.x 或更高版本
  • 至少 50MB 硬盘空间用于思维模型数据库
  • 支持 MCP 协议的客户端应用（VS Code, Cursor, Claude 等）

2

安装"天机"服务器

您可以通过 NPM 全局安装思维模型 MCP 服务器：

npm install -g @thinking-models/mcp-server

或者使用本地项目安装：

npm install @thinking-models/mcp-server

3

启动服务器

全局安装后，可以直接在命令行运行：

thinking-models-server

如果进行了本地项目安装，可以添加到项目的 package.json 脚本中：

{
  "scripts": {
    "start-thinking-models": "thinking-models-server"
  }
}

然后运行：

npm run start-thinking-models

4

配置客户端

启动服务器后，需要配置客户端连接到"天机"服务器。常见的客户端配置：

如果配置正确，应该收到一个包含版本信息的响应。

高级安装选项

自定义端口配置

默认情况下，服务器会监听标准输入/输出与客户端通信。如果需要网络模式，可以设置环境变量：

# Windows 命令行
set THINKING_MODELS_PORT=3456
thinking-models-server

# Linux/macOS 终端
THINKING_MODELS_PORT=3456 thinking-models-server

Docker 部署

使用 Docker 容器部署服务器：

# 构建 Docker 镜像
docker build -t thinking-models-server .

# 运行容器
docker run -p 3456:3456 thinking-models-server

从源码安装

如果您想从源码安装并可能进行自定义修改：

# 克隆仓库
git clone https://github.com/lanyijianke/thinking_models_mcp.git
cd thinking_models_mcp

# 安装依赖
npm install

# 编译代码
npm run build

# 启动服务器
npm start

自定义模型配置

您可以添加自定义思维模型到数据目录：

# 模型文件存放位置
# 全局安装: ~/.thinking_models_mcp/thinking_models_db/{zh|en}/
# 本地安装: ./thinking_models_db/{zh|en}/

# 添加自定义模型示例
echo '{
  "id": "my_custom_model",
  "name": "我的自定义模型",
  "definition": "模型定义...",
  "category": "决策思维",
  "purpose": "模型用途..."
}' > ./thinking_models_db/zh/my_custom_model.json

常见安装问题

可能的原因和解决方案：

  • 端口冲突：如果使用网络模式，检查端口是否被其他应用占用
  • 客户端配置错误：确认客户端中配置的连接 URL 或命令路径正确
  • 防火墙限制：检查防火墙是否允许指定端口的通信

尝试重启服务器并使用默认配置，先确认基本连接是否可用。

可能的原因和解决方案：

  • 文件权限问题：确保模型目录有读取权限
  • 模型文件格式错误：检查自定义模型的 JSON 格式是否正确
  • 安装不完整：尝试重新安装服务器以确保所有依赖和数据文件完整

检查服务器日志以获取更详细的错误信息。

可能的原因和解决方案：

  • Node.js 版本过低：升级 Node.js 到支持的版本（16.x 或更高）
  • 依赖冲突：在一个干净的环境中重新安装

使用 nvm（Node Version Manager）可以轻松切换 Node.js 版本来测试兼容性。
