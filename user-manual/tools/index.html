天机工具概览

"天机"提供一套完整的思维工具阶梯，从基础认知到高阶思维，帮助您在每一个思维能力层次找到适合的工具支持。

工具层级架构

"天机"的工具设计遵循"由简入繁、循序渐进"的原则，构成了一套完整的思考阶梯，帮助用户从基础认知到高阶思维的全面提升。工具一共分为三层：

基础层

提供对于整个MCP服务器版本、状态、学习系统等信息的统计支持工具，同时"天机"也预设了中英文两库的思维模型文件，帮助大家快速上手。

服务层

主要为应用入口提供必要的帮助，比如按不同维度检索模型、对现有模型进行删改，和增加新模型等功能。这算得上是天机内部的"工具"。

应用层

常用的"应用类"工具，这些工具从问题推荐到模型泛化都有涉及，可以满足在多数场景下的实际需求。

思考能力阶梯

最基础的工具层，帮助用户开始思考过程：

  • 探索类 模型列表浏览 (list-models)： 系统化展示可用模型，适合探索式学习
  • 探索类 关键词模型搜索 (search-models)： 最直接的模型获取方式，快速检索相关思维模型
  • 问题解决类 问题导向推荐 (recommend-models-for-problem)： 分析场景，智能推荐模型组合，适合不确定使用哪种模型的场景
  • 系统与学习类 新手入门指南 (get-started-guide)： 为初次使用者提供系统性引导，降低学习门槛

这一层工具操作简单，门槛低，是用户接触"天机"的第一步，特别适合思维模型初学者或特定问题急需解决方案的场景。

在获取基础模型后，这一层工具帮助用户更深入地理解和应用：

  • 探索类 获取模型详情 (get-model-info)： 深入学习模型的原理和应用方法
  • 探索类 相关模型发现 (get-related-models)： 提供关联探索，扩展思考维度，构建思维模型网络
  • 系统与学习类 知识缺口检测 (detect-knowledge-gap)： 识别思考中的盲点，确保全面性
  • 探索类 获取模型分类 (get-categories)： 帮助用户建立思维工具体系认知

这一层工具帮助用户从"知道有这个工具"到"会用这个工具"，是从理论到实践的重要桥梁。

当用户熟悉了基本模型和应用后，这一层工具帮助提升思考质量：

  • 问题解决类 假设生成与验证 (generate-validate-hypotheses)： 系统性生成并验证假设，培养科学思维
  • 问题解决类 推理过程解释 (explain-reasoning-process)： 可视化展示思考逻辑链路，加深对思考机制的元认知
  • 问题解决类 交互式推理引导 (interactive-reasoning)： 提供实时指导和反馈，引导逐步深入思考
  • 系统与学习类 模型使用统计 (get-model-usage-stats)： 帮助用户了解自己的思考偏好和模式

这一层工具不仅关注结果，更关注思考过程本身的质量，帮助用户培养更严谨、更全面的思考习惯。

这是最高级的工具层，实现真正的个性化思考：

  • 创建类 组合化创新模型 (emergent-model-design)： 融合现有模型创造新思考框架，实现创新突破
  • 创建类 思维模型创建 (create-thinking-model)： 用户自主定义新的思考框架，实现知识创造与分享
  • 创建类 思维模型更新 (update-thinking-model)： 修改完善现有思维模型，持续迭代优化个人知识体系
  • 创建类 思维模型删除 (delete-thinking-model)： 管理和优化个人模型库，确保资源高效配置
  • 系统与学习类 用户反馈记录 (record-user-feedback)： 记录使用体验，为系统持续优化提供数据基础

这一层工具帮助用户从"使用标准化工具"过渡到"创造个性化工具"，实现思维模式的个人化提升。

工具的协同效应

在设计"天机"的工具组成时，我们特别注重工具之间的协同效应与层级贯通。用户可能从第一层的问题导向推荐入手，通过第二层的获取模型详情深入学习，在第三层通过交互式推理引导应用模型，最后在第四层通过组合化创新模型创造个性化解决方案。这种跨层级的工具链条使思考过程更加流畅自然，各工具之间相互补充、相互强化。

更重要的是，这些按能力层次组织的工具，在功能上分属于探索类、问题解决类、创建类和系统与学习类四大类别，但对用户而言，它们通过自然语言交互无缝连接，使用户能够专注于思考本身，而非工具的技术分类。
