创建类工具

创建类工具是"天机"的个性化引擎，赋予用户定制、创建和管理自己专属思维模型的能力。这些工具能将用户从标准思维工具的使用者转变为创造者，构建真正贴合个人需求的思考框架。

组合化创新模型 (emergent-model-design)

融合现有模型创造新框架

功能描述

emergent-model-design 工具通过组合现有思维模型的关键概念和特性来创建新的思维模型，实现创新的思考框架。这种方法基于认知嫁接原理，能够产生超越单一模型的涌现特性，解决更加复杂和特定的问题。

参数说明

参数名 类型 必填 默认值 说明
source_model_ids 字符串数组 是 - 用于组合的源思维模型ID列表，最少2个，最多10个
target_model_id 字符串 是 - 新模型的唯一标识符，使用小写字母和下划线
target_model_name 字符串 是 - 新模型的名称
design_goal 字符串 是 - 设计目标和用途描述
category 字符串 否 - 新模型的主要分类（可选）
connection_description 字符串 否 - 描述源模型是如何组合的（可选）
lang 字符串 否 "zh" 语言代码，支持 "zh"(中文) 或 "en"(英文)

使用示例

// 创建战略思维与系统思维的融合模型
{
  "source_model_ids": ["swot_analysis", "feedback_loop", "mental_models"],
  "target_model_id": "adaptive_strategy_system",
  "target_model_name": "适应性战略系统模型",
  "design_goal": "设计一个将战略思维与系统思维结合的模型，用于在快速变化的环境中制定自适应战略。",
  "category": "战略与系统思维",
  "connection_description": "将SWOT分析的结构化评估与反馈循环的动态调整机制结合，通过心智模型框架增强认知深度。",
  "lang": "zh"
}

// 创建创新与决策融合模型
{
  "source_model_ids": ["first_principles_thinking", "decision_matrix", "inversion_thinking"],
  "target_model_id": "innovative_decision_framework",
  "target_model_name": "创新决策框架",
  "design_goal": "建立一个结合第一性原理、决策矩阵和逆向思维的框架，用于在高不确定性环境中做出创新性决策。",
  "lang": "zh"
}

实际应用场景

  • 创建特定行业或领域的专用思维框架
  • 应对跨领域复杂问题的综合思考方法设计
  • 基于团队已有经验构建组织专属思维工具
  • 学术研究中整合多种理论框架形成新视角
提示： 选择互补而非重复的源模型通常能产生更有价值的组合模型。明确的设计目标和应用场景描述有助于系统更准确地融合关键元素。创建后，可以使用 update-thinking-model 工具进一步完善新模型。

思维模型创建 (create-thinking-model)

自定义思考框架

功能描述

create-thinking-model 工具允许用户从零开始定义和创建全新的思维模型，包括基本定义、目的、应用场景、可视化内容等。这是"天机"最强大的个性化工具之一，能够将个人独特的思考方式形式化为可复用的框架。

参数说明

参数名 类型 必填 默认值 说明
id 字符串 是 - 模型的唯一标识符，使用小写字母和下划线
name 字符串 是 - 模型的名称
definition 字符串 是 - 模型的简明定义
purpose 字符串 是 - 模型的主要目的和使用场景
category 字符串 是 - 模型的主要分类
prompt 字符串 否 - 详细的提示词/角色扮演指南（可选）
example 字符串 否 - 模型使用的简短示例（可选）
limitations 对象数组 否 - 模型的局限性（可选）
table_visualizations, bar_chart_visualizations, flowchart_visualizations, comparison_table_visualizations, list_visualizations 对象数组 否 - 各类可视化内容（可选）
lang 字符串 否 "zh" 语言代码，支持 "zh"(中文) 或 "en"(英文)

使用示例

// 创建基本思维模型
{
  "id": "resource_allocation_matrix",
  "name": "资源分配矩阵模型",
  "definition": "一种结构化分析框架，用于在有限资源条件下评估不同项目或投资机会的价值与成本，帮助做出最优的资源分配决策。",

实际应用场景

  • 将个人积累的思考方法形式化为可共享的模型
  • 为特定领域或行业创建专属思维框架
  • 研究人员将独特研究成果转化为实用思维工具
  • 教育工作者创建教学专用的思考框架
提示： 虽然只有少数几个字段是必填的，但提供更完整的信息（如示例、可视化、常见问题等）会使模型更加实用。创建后，您可以通过 get-model-info 工具查看模型，并通过 update-thinking-model 工具进行迭代完善。

思维模型更新 (update-thinking-model)

迭代完善已有模型

功能描述

update-thinking-model 工具允许用户修改和完善现有的思维模型，包括更新任何字段、添加新内容或优化表达。此工具与 create-thinking-model 的主要区别是它保留了模型的演化历史，实现思维模式的迭代更新而非重新创建。

参数说明

参数名 类型 必填 默认值 说明
model_id 字符串 是 - 要更新的模型ID
name, definition, purpose, category, 等 各类型 否 - 任何需要更新的字段（与create-thinking-model相同）
lang 字符串 否 "zh" 语言代码，支持 "zh"(中文) 或 "en"(英文)

使用示例

// 更新模型的基本信息
{
  "model_id": "resource_allocation_matrix",
  "definition": "一种多维度评估框架，用于在资源受限条件下系统化分析不同项目或投资机会的价值、风险与资源需求，支持最优资源分配决策。",
  "subcategories": ["资源管理", "投资组合分析"],
  "lang": "zh"
}

// 添加可视化内容
{
  "model_id": "resource_allocation_matrix",
  "table_visualizations": [
    {
      "title": "资源分配评估矩阵",
      "description": "用于评估项目价值与资源需求的决策表",
      "headers": ["项目", "战略价值(1-10)", "预期回报(1-10)", "资源需求(1-10)", "风险水平(1-10)", "综合评分"],
      "rows": [
        ["项目A", "8", "7", "5", "4", "高"],
        ["项目B", "6", "9", "8", "6", "中"],
        ["项目C", "9", "5", "3", "7", "高"],
        ["项目D", "4", "6", "2", "3", "中"]
      ]
    }
  ],
  "lang": "zh"
}

实际应用场景

  • 基于实际使用体验完善已创建的模型
  • 添加新的可视化或示例丰富模型内容
  • 根据新的见解或研究成果更新模型
  • 修正模型中的不准确或过时内容
提示： 只需要提供需要更新的字段，未提供的字段将保持不变。这使得您可以进行小型、增量的模型优化，而不必每次都重新提交完整模型。定期更新和完善模型可以显著提升其实用性和准确性。

思维模型删除 (delete-thinking-model)

管理个人模型库

功能描述

delete-thinking-model 工具用于从系统中移除不再需要的思维模型。这有助于整理个人模型库，确保只保留高质量、实用的模型，提高思维工具的使用效率。

注意： 模型删除是不可逆操作，请确认无误后再执行。系统预设的基础模型不能被删除。

参数说明

参数名 类型 必填 默认值 说明
model_id 字符串 是 - 要删除的模型ID
confirm 布尔值 是 - 确认删除操作，必须设为true
lang 字符串 否 "zh" 语言代码，支持 "zh"(中文) 或 "en"(英文)

使用示例

// 删除模型
{
  "model_id": "resource_allocation_matrix",
  "confirm": true,
  "lang": "zh"
}

实际应用场景

  • 移除已被更好模型替代的旧模型
  • 清理实验性或测试性质的模型
  • 移除不再适用于当前环境的模型
  • 重新组织个人模型库，提高使用效率
提示： 在删除模型前，请确认是否有依赖于此模型的其他模型或推理过程。如果不确定是否还需要某个模型，可以考虑先使用get-model-usage-stats查看其使用情况，或者保留但添加废弃标记，而不是直接删除。

创建个性化思维模型的最佳实践

要创建真正有价值的个性化思维模型，以下是一些建议的实践：

  1. 先研究后创造 - 使用探索类工具深入了解已有的相关模型，避免重复创造
  2. 从问题出发 - 基于真实的问题需求创建模型，而不是为了创新而创新
  3. 渐进式发展 - 先用create-thinking-model创建基本版本，然后通过update-thinking-model不断迭代完善
  4. 组合优于发明 - 对于复杂问题，使用emergent-model-design组合现有模型往往比从零创建更有效
  5. 重视可视化 - 添加图表、表格等可视化内容能显著提升模型的可理解性和实用性

记住，最有价值的思维模型是那些能够实际应用于解决问题、产生洞见的模型。不断使用、反馈和优化才能让您的个性化思维模型真正成熟。
